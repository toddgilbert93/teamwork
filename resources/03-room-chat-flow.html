Room Group Chat — Full Prompt Flow

POST /api/room/chat — Multiple sequential Anthropic API calls, one per responding persona

Phase 1 Data Gathering
1
Save user message to Supabase
Inserted into room_messages with persona_id: null, role: 'user'
2
Fetch all personas
All personas ordered by sort_order. Each has: system_prompt, memory_summary, name, emoji
3
Fetch room message history
All room_messages with joined persona data. Old contaminated responses are cleaned (truncated at first sign of another persona's name). Excludes the just-inserted user message.
Phase 2 Persona Selection
With @-mentions
If the user message contains @Name, only those mentioned personas respond (filtered through active/unmuted list).
Without @-mentions
Active (non-muted) personas are shuffled randomly, then the first 3 are selected (MAX_INITIAL_RESPONDERS = 3).
Phase 3 Initial Response Round
FOR EACH selected persona (sequential, with 800–1500ms delays)
A
Build system prompt via buildRoomSystemPrompt()
See detailed payload below. Includes identity, memory, group rules, and pass/force instructions.
B
Build messages via buildAnthropicMessages()
Merges room history + prior turn responses into the user message.
Applied through applyRoomWindow() to fit within 60,000 tokens.
C
Call Anthropic API (streaming)
anthropic.messages.stream() — one call per persona
D
Clean response + handle "pass"
cleanPersonaResponse() strips own-name prefixes, XML tags, and truncates at other persona names.
If response is "pass" and not all others passed → skip this persona.
If ALL personas passed → retry last persona with forceRespond=true (another API call!).
Anthropic API Call — Per Persona in Room
Request Parameters
model: "claude-sonnet-4-20250514" max_tokens: 300 // much shorter than 1:1 temperature: 0.3–0.9 // per persona name system: ⬇ constructed system prompt messages: ⬇ room history + turn context
system (string) — buildRoomSystemPrompt()
## Your Identity {persona.system_prompt} ## Private Memory // only if memory_summary exists Here is what you know about the user from your private conversations: {persona.memory_summary} ## Group Discussion Context You are in a group discussion with the user and these other companions: 🌞 Sol, 🦖 Rex, 🔮 Mira. IMPORTANT: You are ONLY {persona.name}. Respond ONLY as yourself. NEVER generate text on behalf of other personas. Do NOT prefix your response with your name or anyone else's name. Just write your response directly — attribution is handled by the system. Keep your response very short — 1 to 2 sentences max. Do NOT agree with or rephrase what another persona just said. If you agree, pass. Only respond if you have a genuinely different angle. You can reference other personas by name when responding to their points. If you have nothing meaningful to add, respond with just "pass". // If forceRespond=true, replaces last line with: You must respond to this message — do not pass. // If isFollowUp=true, adds: ## Follow-Up Instructions Respond to or build on something one of the other personas just said. Do not repeat your earlier point. One sentence only.
messages (array) — buildAnthropicMessages() + applyRoomWindow()
// Room history (consecutive same-role messages merged) [ { role: "user", content: "earlier user message" }, { role: "assistant", content: "Sol response\n\nRex response" }, { role: "user", content: "another user message" }, ... { role: "user", content: "current message + turn context ⬇" } ] // The final user message includes prior turn responses: "What do you think about AI?" [Other companions have already responded to this message:] Sol: "I think AI is fascinating because..." Rex: "The practical applications are what interest me..." [It is now your turn. Write only your own response.] // All windowed to fit within 60,000 tokens
Phase 4 Follow-Up Round (50% chance)
Conditions: No @-mentions were used, at least 2 personas responded (didn't pass), and Math.random() < 0.5.
Selection: 1 persona chosen from initial responders, preferring someone other than whoever spoke last.
System prompt: Same as initial round but with isFollowUp=true and forceRespond=true.
This adds 1 more streaming API call.
API calls per user message in Room Chat:
• Minimum: 1 (1 persona selected, responds without passing)
• Typical: 3 (3 personas each make 1 call)
• With follow-up: 4 (3 initial + 1 follow-up)
• Maximum: ~5-6 (3 initial + forced retry if all pass + follow-up)
• All calls use max_tokens: 300 (vs 4096 in 1:1 chat)
