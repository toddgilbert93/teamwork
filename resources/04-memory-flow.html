Memory Summarization — Prompt Flow

generateMemorySummary() → client.messages.create() — The third and final Anthropic API call pattern

Automatic
Background trigger from 1:1 chat
When applyWindowToMessages() trims more than 20 messages, a fire-and-forget call to generateMemorySummary(trimmed, persona.memory_summary) is triggered.

This runs in the background — the chat response is not blocked by it.
Manual
POST /api/memory with action: "summarize"
Fetches all messages for the persona (not just trimmed ones) and generates a summary.

This is a synchronous request — the response waits for summarization to complete.
1
Format messages as text
Each message → "User: {content}" or "Assistant: {content}"
All joined with double newlines.
2
Build prompt from template
The SUMMARIZE_PROMPT template is filled with:
{existingMemory} → current persona.memory_summary or "None"
{messages} → the formatted message text
3
Call Anthropic API (synchronous, not streaming)
client.messages.create() — waits for full response.
4
Save to Supabase
Updates personas.memory_summary with the generated summary text.
Anthropic API Call — client.messages.create()
Request Parameters
model: "claude-sonnet-4-20250514" max_tokens: 1024 messages: [ { role: "user", content: "...prompt below..." } ] // NOTE: No system prompt. No temperature. No streaming.
The single user message content (full prompt)
You are a memory management system. Create a concise summary of this conversation that captures the essential context needed to continue naturally. Previous memory summary (if any): {existingMemory or "None"} Recent messages to incorporate: User: Hey, I'm working on a new project... Assistant: That sounds exciting! Tell me more... User: It's a web app for tracking... ... Create an updated summary that: 1. Preserves key facts, decisions, and context from both the existing summary and new messages 2. Notes the user's preferences, goals, and ongoing projects 3. Tracks any commitments, action items, or decisions made 4. Captures the emotional tone and relationship dynamics 5.
