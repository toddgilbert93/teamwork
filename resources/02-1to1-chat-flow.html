1:1 Persona Chat — Full Prompt Flow

POST /api/chat → anthropic.messages.stream() — Exactly what gets sent to Anthropic and how it's constructed

1
Browser sends message
useChat hook calls POST /api/chat with:
{ persona_id: "uuid", message: "user's text" }
2
Fetch persona from Supabase
Retrieves: system_prompt, memory_summary, name
The persona's name determines its temperature setting.
3
Fetch full message history from Supabase
All messages for this persona, ordered by created_at ASC.
New user message is appended to this array in memory.
4
Apply sliding window
applyWindowToMessages() keeps only recent messages that fit within ~71,904 tokens.
Budget = 80,000 (target) − 4,000 (system buffer) − 4,096 (max response)
Token estimation: Math.ceil(text.length / 3.5) + 4 per message

Returns: { windowed: Message[], trimmed: Message[] }
5
Build system prompt
If persona has a memory_summary, it's prepended to the system prompt:

## Conversation Memory
The following is a summary of earlier conversations...
{memory_summary}
---
## Your Identity
{persona.system_prompt}

If no memory exists, just persona.system_prompt alone.
6
Format messages for API
Each windowed message is mapped to: { role: "user"|"assistant", content: string }
No additional formatting — raw message content is passed through.
Anthropic API Call — anthropic.messages.stream()
Request Parameters
model: "claude-sonnet-4-20250514" max_tokens: 4096 temperature: 0.3 // Sol=0.3, Rex=0.5, Mira=0.7, Mean guy=0.9 system: ⬇ see below messages: ⬇ see below
system (string)
// WITH memory: ## Conversation Memory The following is a summary of earlier conversations with this user: {persona.memory_summary} --- ## Your Identity {persona.system_prompt} // WITHOUT memory: {persona.system_prompt}
messages (array) — sliding window, most recent that fit in ~71,904 tokens
[ { role: "user", content: "oldest retained message..." }, { role: "assistant", content: "persona response..." }, { role: "user", content: "..." }, { role: "assistant", content: "..." }, ... { role: "user", content: "the new user message" } // always last ]
7
Stream response back via SSE
Response tokens are streamed as data: {"type":"text_delta","text":"..."}
On completion: data: {"type":"message_complete","message_id":"..."}
Full response is saved to Supabase messages table.
8
Background memory summarization (conditional)
Only if trimmed.length > 20 — fires a background (non-blocking) call to
generateMemorySummary(trimmed, persona.memory_summary)
This makes a separate synchronous API call — see Memory Summarization diagram.
Result is saved to personas.memory_summary.
Key insight: A single user message in 1:1 chat produces 1 streaming API call (always), plus potentially 1 additional synchronous API call for memory summarization (only when >20 messages are trimmed from the context window). So 1–2 API calls total per user message.
